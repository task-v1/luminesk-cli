# Repository Providers

Luminesk uses an online JSON registry to discover and download server engines. Each engine has an associated **Provider Type** which defines how the CLI queries remote servers to resolve and download the latest version of the core JAR file.

Luminesk supports three provider types: **Maven**, **Jenkins**, and **GitHub Release**.

---

## 1. Maven Provider

The Maven provider resolves artifacts hosted on standard Apache Maven repositories (such as the OpenCollab Maven repository, PowerNukkitX Maven, or the Lumi repo).

### Mechanics:
- **Metadata Inspection**: Luminesk requests the `maven-metadata.xml` file located at the artifact directory.
- **Snapshot Resolution**: For snapshot releases (e.g., `1.0.0-SNAPSHOT`), it parses the XML to extract the latest snapshot timestamp and build number, converting a version to an absolute download URL (e.g., `nukkit-1.0.0-20260610.123456-1.jar`).
- **Safety Limits**: XML parsing is protected by file size safety limits to prevent denial-of-service (DoS) from bloated metadata payloads.

---

## 2. Jenkins Provider

The Jenkins provider downloads artifacts generated by continuous integration (CI) jobs running on Jenkins servers (e.g. Nukkit-MOT).

### Mechanics:
- **API Resolution**: Luminesk queries the Jenkins REST API endpoint (`/api/json`) for the target job.
- **Artifact Matching**: It parses the build response to locate successful build numbers and scans the compiled assets checklist.
- **Classifier Filtering**: If a classifier is specified (like `all` or `shaded`), it filters down to the exact JAR file, resolving the download URL directly to the Jenkins artifact storage path.

---

## 3. GitHub Release Provider

The GitHub Release provider fetches compiled binaries published on GitHub repository release pages.

### Mechanics:
- **API Querying**: Queries the official GitHub Releases API (`/api.github.com/repos/.../releases/latest`).
- **Asset Filtering**: Scans the assets list of the release and uses regex patterns (or exact matches) to download the target file.
- **Rate-Limiting Protection**: If GitHub API rate limits are hit, Luminesk prompts users to set a `GITHUB_TOKEN` environment variable to authenticate requests and lift the limit.
